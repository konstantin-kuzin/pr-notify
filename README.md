# PR Notify

## Назначение

`PR Notify` — Chrome-расширение для мониторинга pull requests на странице Azure DevOps.


Основной сценарий: пользователь видит количество активных PR своей команды прямо на иконке расширения и может открыть список PR в popup.

Расширение ориентировано на работу в уже авторизованной сессии TFS в браузере Chrome.

## Функциональность

Текущая реализация поддерживает следующий набор возможностей:

- автоматически проверяет список pull requests каждые 10 минут
- выполняет первую проверку после установки, запуска браузера и старта service worker
- умеет запускать ручное обновление из popup по кнопке
- показывает badge на иконке расширения с количеством найденных PR
- открывает popup со списком pull requests по клику на иконку расширения
- показывает в popup:
  - аватар автора
  - название PR
  - автора PR
  - время с момента создания PR до проверки
- подсвечивает PR с момента создания которых прошло больше 48 часов
- отображает tooltip с описанием PR, если описание присутствует в источнике данных
- помечает PR бейджем `ТЕХ ПР`, если в описании найден соответствующий маркер
- отправляет системное уведомление, если при очередной успешной проверке появились новые PR
- при ошибке загрузки или разбора страницы скрывает badge с количеством, переключает иконку расширения на error-вариант и показывает текст ошибки в popup
- сохраняет последнее успешное состояние и не очищает список при временной ошибке загрузки

## Какие PR попадают в список

В выборку попадают только элементы, которые одновременно удовлетворяют следующим условиям:

- PR находится в запросе `Assigned to my teams`
- PR относится к вкладке `Active`
- PR ещё не считается просмотренным для целевого reviewer/group reviewer

После отбора список сортируется по дате создания PR:
- более старые PR находятся выше
- более новые PR находятся ниже

## Принцип работы

### 1. Получение данных

Фоновый скрипт [background.mjs](/background.mjs) выполняет `fetch` целевой страницы с `credentials: "include"`, поэтому использует текущую браузерную авторизацию пользователя.

Расширение извлекает JSON из блока:

- `<script id="dataProviders" type="application/json">`

Это позволяет работать без открытия самой страницы в активной вкладке.

### 2. Разбор страницы

Парсер в [parser.mjs](/parser.mjs) работает с provider `ms.vss-code-web.prs-list-data-provider` и выполняет следующие шаги:

1. Находит список запросов `queries`
2. Выбирает нужный запрос: `Assigned to my teams` или `Assigned to me`
3. Берёт идентификаторы PR из `queryResults`
4. Находит подробные данные PR в `pullRequests`
5. Отфильтровывает только `Active` элементы по голосам reviewer
6. Преобразует данные в внутреннюю структуру:
   - `id`
   - `title`
   - `author`
   - `avatarUrl`
   - `createdAt`
   - `description`
   - `url`
7. Сортирует итоговый список по возрастанию даты создания

### 3. Хранение состояния

Состояние расширения сохраняется в `chrome.storage.local` под ключом `prState`.

Сохраняются:
- текущий список PR
- количество PR
- время последней проверки
- время последней успешной проверки
- источник последнего запуска проверки
- текст последней ошибки
- список ID предыдущей успешной выборки

Этот список ID используется для определения новых PR и показа уведомлений.

### 4. Обновление badge и уведомлений

После каждой успешной проверки background:

- пересчитывает количество PR
- обновляет badge у иконки расширения
- сравнивает текущие ID с предыдущими
- если появились новые PR, показывает системное уведомление

Если список пустой, badge очищается.

Если проверка завершилась ошибкой или на странице не найден целевой блок:

- в состояние сохраняется текст ошибки в `lastError`
- badge с количеством очищается, чтобы не показывать устаревшее число PR
- иконка переключается на error-вариант
- popup продолжает использовать последний успешно сохранённый список PR и одновременно показывает сообщение об ошибке
- при следующем запуске service worker или перед новым обновлением состояние ошибки сначала восстанавливается из сохранённого состояния

### 5. Работа popup

Popup реализован в [popup.html](/popup.html), [popup.css](/popup.css) и [popup.mjs](/popup.mjs).

При открытии popup:

- читается последнее сохранённое состояние из `chrome.storage.local`
- рисуется список PR
- показывается время последней проверки
- подписывается обработчик на изменения `chrome.storage.onChanged`, чтобы popup обновлялся при новой проверке

Popup ограничен по высоте половиной доступной высоты экрана. Прокрутка появляется только тогда, когда содержимое превышает этот предел.

  - извлечение и нормализация PR из `dataProviders`

  - стили popup
- [popup.mjs](/popup.mjs)
  - рендер списка, ручное обновление, tooltip, форматирование возраста PR

## Разрешения расширения

Расширение использует следующие разрешения:

- `alarms` — периодический запуск проверки
- `storage` — сохранение текущего состояния
- `notifications` — показ уведомлений о новых PR
- `host_permissions` для `https://hqrndtfs.avp.ru/*` — доступ к странице TFS

## Ограничения и допущения

- расширение зависит от активной авторизации пользователя в TFS/Azure DevOps
- источник данных — внутренняя структура `dataProviders`; при существенном изменении разметки страницы или формата provider потребуется обновление парсера
- относительное время сейчас считается от `creationDate`, потому что в `dataProviders` нет отдельного времени `Updated` для PR
- popup показывает состояние последней сохранённой проверки, а не выполняет парсинг самостоятельно

## Проверка и разработка

```bash
node --check background.mjs
node --check parser.mjs
node --check popup.mjs
```

Для запуска в Chrome расширение загружается как unpacked extension через `chrome://extensions`.
